Maritime AI Surveillance Platform - Project Documentation
This document explains the architecture, features, and key components of the Maritime AI Surveillance Platform. The project is divided into three main components: an AI Microservice (ai_service), a Node.js Backend (backend), and a React Frontend (frontend).

1. Architecture Overview
The system processes live AIS (Automatic Identification System) data to track vessels in real-time. It analyzes this data for potential threats using both heuristic rules and a machine-learning model. Detected anomalies generate alerts that are broadcasted to a frontend dashboard via WebSockets.

Frontend: React (Vite) application providing the command center UI.
Backend: Node.js (Express & Socket.io) handling data ingestion, database storage, and real-time client updates.
AI Service: Python (Flask) microservice running a Scikit-learn Isolation Forest model for anomaly detection.
Database: MongoDB used to store vessel histories, active alerts, and tracking data.
2. Component Details & Features
2.1 AI Microservice (ai_service)
This Python application acts as an independent microservice dedicated to predicting vessel threat levels based on machine learning and heuristic rules.

Key Files & Features:

app.py: A Flask server exposing a /predict endpoint.
Machine Learning (Isolation Forest): Trains a model on synthetic "normal" vessel data (speed, heading, latitude, longitude, and vessel type) in the Gulf region. The model outputs a decision score to determine how "anomalous" a vessel's behavior is compared to the norm.
Heuristics & Threat Scoring:
Proximity to Ports: Calculates Haversine distance to major ports (Houston, Galveston, Miami, Seattle).
Speed Anomalies: Increases threat scores for excessive speeds (e.g., > 30 knots) or suspicious speeds based on vessel type.
Zone Threat: Flags vessels loitering near ports or traveling uncharacteristically fast in restricted zones.
Vessel of Interest (VOI) Level: Categorizes the vessel into one of five levels: Normal, Monitor, Vessel of Interest, High Risk, or Critical.
2.2 Backend Node.js Server (backend)
The backend is the core router of the application. It ingests data, evaluates immediate rule-based threats, stores history, and pushes updates to the client.

Key Files & Features:

server.js: The main entry point. Sets up the Express API, connects to MongoDB via Mongoose, and initializes the Socket.io WebSocket server.
Data Ingestion (aisStream.js & aisSimulator.js):
Connects to the real-time aisstream.io WebSocket to receive live ship positions worldwide.
Includes a simulator fallback to generate fake AIS data if the API key is not provided.
Threat Engine (services/threatEngine.js):
Acts as a bridge to the ai_service. It sends vessel data to the Python microservice and receives the machine-learning threat score.
Implements localized checks like Restricted Zone Entry, Speed Anomalies, and Abrupt Heading Changes.
Generates database Alert records and emits socket events to the frontend when anomalies occur.
SAR Module (services/sar/sarScheduler.js & routes/sarRoutes.js):
Simulates automated Search and Rescue (SAR) tracking by correlating "Dark Vessels" (vessels detected via SAR imagery but not transmitting AIS).
Database Models (models/): Defines MongoDB schemas for Alert, Track, and Vessel.
REST API (routes/): Exposes historical vessel data, statistics, and alert clearing endpoints to the frontend.
2.3 Frontend Dashboard (frontend)
The frontend is a rich, interactive command center built using React and Vite. It utilizes WebSockets for real-time interactivity.

Key Files & Features:

App.jsx: The main component managing application state, WebSocket connections, active pages, and data fetching.
Live Vessel Map (components/VesselMap.jsx): Uses Leaflet.js to plot vessels on a geographical map in real-time. Features include custom markers based on threat level and vessel clustering.
Real-time Dashboard & Stats (components/StatsCards.jsx & components/Sidebar.jsx): Displays aggregate statistics like active vessels, critical alerts, and average fleet speed. Provides filtering capabilities by MMSI, vessel type, and threat level.
Alert Panel (components/AlertPanel.jsx): A scrolling feed of real-time alerts generated by the backend's Threat Engine.
Vessel Details & Replay Mode (components/VesselDetails.jsx): Selecting a vessel shows its metadata and current threat score. A "Replay Mode" allows operators to fetch the historical tracking coordinates from the database and animate the vessel's past movements on the map.
SAR Dashboard (pages/SarDashboard.jsx): A dedicated view for correlating and analyzing Search and Rescue imagery against known AIS tracks.
3. Data Flow Summary
Ingestion: backend receives a live AIS packet from aisstream.io.
Analysis: backend passes the data to threatEngine.js.
ML Evaluation: threatEngine.js makes an HTTP POST request to the ai_service to get the Isolation Forest anomaly score.
Alert Generation: threatEngine.js combines the ML score with local rules. If a threshold is crossed, it creates an Alert in MongoDB.
Storage: The vessel's new state and tracking coordinates are saved in MongoDB.
Broadcast: backend uses Socket.io to emit vessel-update and vessel-alert events.
Display: The frontend React app receives the socket events and instantly updates the map, alert panel, and statistics widgets without requiring a page refresh.
